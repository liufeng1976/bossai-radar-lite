# BossAI Radar Lite 每日线索跟进工作法

BossAI Radar Lite v0.5 的目标不是自动群发，而是让销售人员每天打开系统后，立即知道：

1. 今天必须联系谁；
2. 为什么现在要联系；
3. 应该发送什么内容；
4. 联系后应推进到哪个阶段；
5. 下一次何时继续跟进。

## 四类跟进队列

| 队列 | 含义 | 处理顺序 |
|---|---|---:|
| OVERDUE | 已超过计划跟进日期 | 最高 |
| TODAY | 今天到期 | 高 |
| UNSCHEDULED | 活跃线索但没有安排下一次跟进 | 中高 |
| UPCOMING | 未来指定天数内到期 | 中 |

`WON` 和 `LOST` 不进入活跃跟进队列。

## 紧迫度排序

系统综合以下因素生成 0–100 的紧迫度：

- 是否逾期或今天到期；
- HOT / WARM / COOL；
- 当前成交阶段；
- 是否计划立即或 30 天内上线；
- 是否已经存在报价。

紧迫度仅用于排序，不会自动修改线索数据，也不会自动发送消息。

## 建议阶段

| 当前状态 | 建议下一状态 |
|---|---|
| NEW | QUALIFIED |
| QUALIFIED | CONTACTED |
| CONTACTED | PROPOSAL |
| PROPOSAL | NEGOTIATION |
| NEGOTIATION | WON |
| WAITLIST | WAITLIST |

点击“应用建议下一步”时，系统会：

- 更新为建议阶段；
- 根据 HOT / WARM / COOL 安排下一次跟进；
- 写入一条系统建议执行记录。

它不会自动向客户发送任何内容。

## 客户话术与后台语言

- 后台的原因、排序和建议动作跟随后台当前语言；
- 真正复制给客户的邮件或消息跟随该线索记录的客户语言；
- 可识别到邮箱时可打开本机邮件客户端；
- 没有邮箱时仍可复制话术，通过微信、短信或其他渠道发送。

## 每日执行顺序

1. 先处理 OVERDUE 中的 HOT；
2. 再处理其他 OVERDUE；
3. 处理 TODAY；
4. 给 UNSCHEDULED 活跃线索安排日期；
5. 检查未来 7 天 UPCOMING；
6. 每次联系后记录结果、更新阶段并安排下一次跟进。

## 导出

管理员可以导出：

- 中文或英文 Markdown 跟进日报；
- 未来 30 天 `.ics` 日历文件。

逾期事件在导入日历时会安排到最近可执行时段，避免生成已经过去的日历事件。

## 安全边界

所有跟进队列、话术、日报和日历接口都属于管理员接口，必须使用 `RADAR_ADMIN_API_KEY`。Lite 不提供自动邮件群发、微信自动触达、第三方 CRM 同步或多人销售权限；这些能力属于商业 Pro 范围。
