# BossAI Radar Lite 商业线索数据说明

BossAI Radar Lite v0.4.0 可以把商业授权申请和 Pro 等待名单保存到当前部署使用的本地 SQLite 数据库。

## 保存的数据

当申请人主动勾选许可确认并提交时，系统可能保存：

- 申请类型；
- 姓名、公司或团队；
- 申请人主动填写的联系方式；
- 团队规模、预算、预计上线时间和部署方式；
- 使用场景与所需能力；
- 界面语言；
- 提交时间和许可确认时间；
- 后续由管理员添加的状态、负责人、报价、跟进时间与活动记录。

## 不保存的数据

Lite 默认不会把以下信息写入线索数据库：

- 访问者原始 IP 地址；
- 浏览器 Cookie；
- 设备指纹；
- 密码；
- 邮件账号凭据；
- 支付卡信息。

IP 只在进程内用于短时提交频率限制，服务重启后即消失。

## 数据位置

线索与雷达证据保存在同一个本地文件中：

```text
data/radar-lite.sqlite
```

该文件被 `.gitignore` 排除，不应上传到 GitHub、公开网盘或发布包。

## 访问控制

- 公开接口只能提交申请，不能读取线索；
- 读取、导出、修改、跟进和删除线索需要 `RADAR_ADMIN_API_KEY`；
- 仅监听 `127.0.0.1` 时，本机管理员页面可直接访问；
- 公网部署必须使用 HTTPS 和强随机管理员密钥；
- 不要把管理员密钥写入网页源码、截图、日志或仓库。

## 去重与限流

- 相同联系方式和申请类型在 24 小时内重复提交时，系统返回原线索编号，不重复创建；
- 默认同一网络来源每小时最多提交 5 次；
- 隐藏蜜罐字段用于拦截简单机器人提交。

## 管理员责任

部署方是线索数据的实际控制者，应根据所在地法律和业务政策：

- 告知申请人数据用途；
- 只收集完成授权评估所需的信息；
- 限制管理员访问；
- 定期备份和清理；
- 在申请人合理要求时导出、更正或删除数据；
- 不把联系人数据用于未披露的营销或转售。

管理员可通过后台删除单条线索，删除会同时清除其全部跟进活动。

## 关闭线索功能

```env
COMMERCIAL_LEAD_CAPTURE_ENABLED=false
COMMERCIAL_LEAD_ADMIN_ENABLED=false
```

关闭入库后，商业申请页仍可使用复制和邮件备份模式。
